URL/API 목록

POKA 시스템은 RESTful API를 기반으로 하며, JSON 형식으로 데이터를 주고받습니다.
모든 API는 HTTPS를 통해 안전하게 통신하며, JWT 토큰을 통한 인증을 지원합니다.

API 기본 정보

🌐 기본 URL

  • API 서버: https://pkapi.ting.ovh
  • 파일 서버: https://pkfile.ting.ovh
  • 업로드 서버: https://upload.ting.ovh

🔐 인증 방식

  • JWT 토큰 - Bearer 방식
  • 토큰 만료: 20분
  • 자동 갱신: 지원
  • 헤더: Authorization: Bearer {token}

📊 응답 형식

  • 성공: HTTP 200 + JSON
  • 오류: HTTP 4xx/5xx + JSON
  • 인코딩: UTF-8
  • Content-Type: application/json

인증 API

구분 URL 메서드 설명
키오스크 로그인 https://pkapi.ting.ovh/user/Kiosk/login POST 키오스크 로그인 인증 및 JWT 토큰 발급

로그인 API 요청/응답 샘플

요청 (Request): POST https://pkapi.ting.ovh/user/Kiosk/login Content-Type: application/json { "username": "kiosk_user", "password": "kiosk_password" } 응답 (Response): HTTP/1.1 200 OK Content-Type: application/json { "success": true, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expiresIn": 1200, "user": { "id": 1, "username": "kiosk_user", "role": "kiosk" } }

API 사용 시 주의사항

🔐 보안

  • 모든 API 호출 시 JWT 토큰 필수
  • 토큰 만료 시 자동 갱신 필요
  • HTTPS 통신만 허용
  • 민감한 정보 로깅 금지

⚡ 성능

  • API 응답 시간: 3초 이내
  • 이미지 업로드: 10MB 이하
  • 동시 요청 제한: 100개
  • 타임아웃: 30초

🔄 재시도

  • 네트워크 오류 시 3회 재시도
  • 재시도 간격: 1초, 3초, 5초
  • 결제 API는 재시도 금지
  • 로그인 실패 시 5분 대기
