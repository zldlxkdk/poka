개발 가이드라인

코딩 표준: 1. 네이밍 컨벤션 - 클래스: PascalCase (예: PaymentService) - 메서드: PascalCase (예: ProcessPayment) - 변수: camelCase (예: orderId) - 상수: UPPER_CASE (예: MAX_RETRY_COUNT) 2. 파일 구조 - 한 파일당 하나의 클래스 - 네임스페이스 사용 - 적절한 폴더 구조 3. 주석 작성 - 모든 public 메서드에 XML 주석 - 복잡한 로직에 인라인 주석 - TODO 주석 사용 4. 오류 처리 - try-catch 블록 사용 - 적절한 예외 타입 선택 - 로깅 포함 Git 워크플로우: 1. 브랜치 전략 - main: 프로덕션 코드 - develop: 개발 브랜치 - feature/*: 기능 개발 - hotfix/*: 긴급 수정 2. 커밋 메시지 - feat: 새로운 기능 - fix: 버그 수정 - docs: 문서 수정 - style: 코드 스타일 - refactor: 리팩토링 - test: 테스트 코드 - chore: 빌드/도구
