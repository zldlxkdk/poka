🔐 JWT 토큰 구조

📋 Header (헤더)

  • alg: HS256 (HMAC SHA256)
  • typ: JWT
  • kid: 키 식별자
  • ver: 토큰 버전

📄 Payload (페이로드)

  • sub: 사용자 ID
  • username: 사용자명
  • role: 사용자 역할
  • iat: 발급 시간
  • exp: 만료 시간

🔏 Signature (서명)

  • 알고리즘: HMAC SHA256
  • 비밀키: 서버 전용 키
  • 검증: 무결성 확인
  • 위변조: 방지

⏰ 토큰 생명주기

  • 발급: 로그인 성공 시
  • 유효기간: 24시간
  • 갱신: 자동 갱신
  • 폐기: 로그아웃 시

🔓 무차별 대입 공격

  • 계정 잠금: 5회 실패 시 30분 잠금

  • IP 차단: 반복 시도 IP 차단

  • 토큰 만료: 24시간 자동 만료
  • 재발급: 의심스러운 활동 시
  • HTTPS: 모든 통신 암호화
  • 세션 추적: 활성 세션 모니터링

💉 SQL 인젝션

  • 파라미터화: Prepared Statements

  • 이스케이프: 특수문자 처리
  • ORM 사용: Entity Framework

🌐 XSS 공격
