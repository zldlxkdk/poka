🗂️ 테이블 상세 구조

테이블명 컬럼명 데이터 타입 제약조건 설명
Users id INT PRIMARY KEY, AUTO_INCREMENT 사용자 고유 식별자
username VARCHAR(50) NOT NULL, UNIQUE 사용자명
role VARCHAR(20) NOT NULL, DEFAULT 'user' 사용자 역할 (admin, user)
Orders id INT PRIMARY KEY, AUTO_INCREMENT 주문 고유 식별자
orderId VARCHAR(100) NOT NULL, UNIQUE 주문 번호 (ORDER_YYYYMMDD_XXX)
amount INT NOT NULL 결제 금액 (원)
paymentType VARCHAR(20) NOT NULL 결제 수단 (cash, linepay, coupon)
status VARCHAR(20) NOT NULL, DEFAULT 'pending' 주문 상태 (pending, completed, failed)
createdAt DATETIME NOT NULL, DEFAULT CURRENT_TIMESTAMP 주문 생성 시간
Uploads id INT PRIMARY KEY, AUTO_INCREMENT 업로드 고유 식별자
uploadId VARCHAR(100) NOT NULL, UNIQUE 업로드 번호 (UPLOAD_XXXXX)
sessionId VARCHAR(100) NOT NULL 세션 식별자
faceImage VARCHAR(255) NULL 얼굴 이미지 URL
backImage VARCHAR(255) NULL 배경 이미지 URL
createdAt DATETIME NOT NULL, DEFAULT CURRENT_TIMESTAMP 업로드 생성 시간

👥 Users 테이블

id: INT (PRIMARY KEY)

username: VARCHAR(50) (UNIQUE)

role: VARCHAR(20) (DEFAULT 'user')

이미지 업로드 정보

📁 파일 구조

🖼️ 이미지 파일

  • 프로필 이미지: /profiles/ 디렉토리
  • 업로드 이미지: /u/{ID}/face, /u/{ID}/back
  • 광고 이미지: /ads/ 디렉토리
  • 로고 이미지: /logos/ 디렉토리

🎨 UI 리소스

  • 스티커: /stickers/ 디렉토리
  • 배경: /backgrounds/ 디렉토리
  • 아이콘: /icons/ 디렉토리
  • 폰트: /fonts/ 디렉토리

⚙️ 설정 파일

  • appsettings.json: 애플리케이션 설정
  • data.json: 정적 데이터
  • ApiEndpoints.json: API 엔드포인트
  • launchSettings.json: 실행 설정

📝 로그 파일

  • 시스템 로그: /logs/system/
  • 오류 로그: /logs/error/
  • 접근 로그: /logs/access/
  • 결제 로그: /logs/payment/

🔗 데이터 관계

📊 1:N 관계

  • Users → Orders: 한 사용자가 여러 주문
  • Users → Uploads: 한 사용자가 여러 업로드
  • Orders → Uploads: 한 주문에 여러 업로드

🔍 인덱스 구조

  • Primary Key: id (모든 테이블)
  • Unique Index: orderId, uploadId
  • Foreign Key: userId (Orders, Uploads)
  • Composite Index: (userId, createdAt)

🗃️ 캐시 구조

  • 메모리 캐시: 자주 사용되는 데이터
  • 파일 캐시: 이미지 및 리소스
  • 세션 캐시: 사용자 세션 정보
  • API 캐시: 외부 API 응답

💾 백업 구조

  • 데이터베이스: 일일 자동 백업
  • 파일 시스템: 주간 증분 백업
  • 설정 파일: 버전 관리
  • 로그 파일: 월별 아카이브
